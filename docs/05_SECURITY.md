## 1. セキュリティ原則

### 1.1 基本方針

TalentSyncは以下のセキュリティ原則に基づいて設計・運用されています。

| 原則 | 説明 |
|------|------|
| **最小権限の原則** | 必要最小限のアクセス権限のみを付与 |
| **多層防御** | 複数のセキュリティ層で保護 |
| **セキュアバイデザイン** | 設計段階からセキュリティを考慮 |
| **ゼロトラスト** | すべてのアクセスを検証 |
| **透明性** | セキュリティ対策を顧客に開示 |

### 1.2 セキュリティ認証

| 認証・規格 | ステータス |
|------------|-----------|
| SOC 2 Type II | 取得予定 |
| ISO 27001 | 取得予定 |
| プライバシーマーク | 取得予定 |

## 2. インフラストラクチャセキュリティ

### 2.1 ホスティング

| 項目 | 詳細 |
|------|------|
| **プラットフォーム** | Vercel（フロントエンド）、Supabase（バックエンド） |
| **データセンター** | AWS東京リージョン |
| **物理セキュリティ** | SOC 2、ISO 27001準拠のデータセンター |

### 2.2 ネットワークセキュリティ

| 対策 | 説明 |
|------|------|
| **TLS暗号化** | すべての通信はTLS 1.3で暗号化 |
| **DDoS保護** | Cloudflare/Vercelによる自動防御 |
| **ファイアウォール** | 不正アクセスの検知・遮断 |
| **エッジネットワーク** | グローバルCDNによる配信 |

### 2.3 証明書管理

- 自動証明書発行（Let's Encrypt）
- 証明書の自動更新
- HSTS（HTTP Strict Transport Security）有効

## 3. アプリケーションセキュリティ

### 3.1 OWASP Top 10対策

| 脆弱性 | 対策 |
|--------|------|
| **インジェクション** | パラメータ化クエリ、入力バリデーション |
| **認証の不備** | Supabase Auth、セッション管理 |
| **機微データの露出** | 暗号化、最小権限アクセス |
| **XXE** | XMLパーサー無効化 |
| **アクセス制御の不備** | RLS、RBAC |
| **セキュリティ設定ミス** | セキュリティヘッダー、デフォルト設定見直し |
| **XSS** | DOMPurify、CSP |
| **安全でないデシリアライゼーション** | 型チェック、バリデーション |
| **既知の脆弱性** | 定期的な依存関係更新 |
| **不十分なログ** | 監査ログ、アラート設定 |

### 3.2 入力バリデーション

すべてのユーザー入力は以下の検証を実施：

```
クライアント側バリデーション
        ↓
APIバリデーション（型、長さ、形式）
        ↓
サニタイズ処理
        ↓
データベース保存
```

### 3.3 セキュリティヘッダー

| ヘッダー | 値 |
|----------|-----|
| `X-Content-Type-Options` | nosniff |
| `X-Frame-Options` | DENY |
| `X-XSS-Protection` | 1; mode=block |
| `Referrer-Policy` | strict-origin-when-cross-origin |
| `Content-Security-Policy` | default-src 'self' |

### 3.4 依存関係管理

- 週次での脆弱性スキャン（npm audit）
- Dependabotによる自動更新PR
- 重大な脆弱性は24時間以内に対応

## 4. データ保護

### 4.1 データ分類

| 分類 | 例 | 保護レベル |
|------|-----|-----------|
| **機密** | 評価コメント、1on1内容 | 暗号化＋アクセス制限 |
| **社外秘** | SF結果、SPI結果 | アクセス制限 |
| **社内** | 氏名、所属チーム | 認証必須 |

### 4.2 暗号化

| 対象 | 方式 |
|------|------|
| **通信** | TLS 1.3 |
| **保存データ** | AES-256（Supabase） |
| **バックアップ** | AES-256 |

### 4.3 データ保持

| データ種別 | 保持期間 |
|------------|----------|
| アクティブデータ | 契約期間中 |
| 退職者データ | 退職後5年 |
| 監査ログ | 3年 |
| バックアップ | 30日 |

### 4.4 データ削除

契約終了時のデータ削除プロセス：

1. 顧客にデータエクスポート機会を提供（30日間）
2. 本番データの完全削除
3. バックアップからの削除（30日以内）
4. 削除完了証明書の発行

## 5. アクセス制御

### 5.1 認証

| 方式 | 詳細 |
|------|------|
| **メール/パスワード** | 標準認証 |
| **マジックリンク** | メールでのワンタイム認証 |
| **SSO（予定）** | SAML 2.0 / OIDC |
| **2FA（予定）** | TOTP / SMS |

### 5.2 パスワードポリシー

| 項目 | 要件 |
|------|------|
| 最小文字数 | 8文字 |
| 複雑性 | 英大文字、小文字、数字を含む |
| 有効期限 | 90日（推奨） |
| 履歴 | 過去5回のパスワード再利用不可 |

### 5.3 行レベルセキュリティ（RLS）

データベースレベルでのアクセス制御：

```sql
-- 例: 評価データのRLSポリシー
CREATE POLICY "evaluations_select" ON evaluations
  FOR SELECT USING (
    -- 管理者は全件閲覧可能
    is_admin()
    OR
    -- マネージャーは自チームのみ
    (is_manager() AND employee_id IN (
      SELECT id FROM employees WHERE team_id = get_user_team_id()
    ))
    OR
    -- 本人のみ
    employee_id = get_current_employee_id()
  );
```

### 5.4 役割ベースアクセス制御（RBAC）

| 権限 | admin | manager | member |
|------|-------|---------|--------|
| 社員一覧閲覧 | ○ | ○ | ○ |
| 社員詳細閲覧 | ○ | ○ | ○ |
| 評価閲覧（全員） | ○ | × | × |
| 評価閲覧（自チーム） | ○ | ○ | × |
| 評価閲覧（自分） | ○ | ○ | ○ |
| 評価編集 | ○ | △ | × |
| SF/SPI入力 | ○ | × | × |
| チーム管理 | ○ | × | × |
| AI再生成 | ○ | × | × |

△: 自チームメンバーのみ

## 6. 監査とログ

### 6.1 監査ログ

以下の操作を記録：

| カテゴリ | 操作例 |
|----------|--------|
| 認証 | ログイン、ログアウト、パスワード変更 |
| データ変更 | 作成、更新、削除 |
| 権限変更 | 権限付与、権限削除 |
| エクスポート | データダウンロード |

### 6.2 ログ項目

| 項目 | 説明 |
|------|------|
| タイムスタンプ | UTC形式 |
| ユーザーID | 操作者の識別子 |
| IPアドレス | 接続元IP |
| 操作種別 | CREATE/READ/UPDATE/DELETE |
| 対象リソース | テーブル名、レコードID |
| 変更内容 | 変更前後の値（機密情報はマスク） |

### 6.3 ログ保管

- 保管期間: 3年
- 改ざん防止: 追記のみ可能
- アクセス制限: システム管理者のみ

## 7. インシデント対応

### 7.1 インシデント分類

| レベル | 定義 | 対応時間 |
|--------|------|----------|
| **Critical** | データ漏洩、サービス停止 | 1時間以内 |
| **High** | セキュリティ侵害の疑い | 4時間以内 |
| **Medium** | 脆弱性発見 | 24時間以内 |
| **Low** | 軽微なセキュリティ問題 | 72時間以内 |

### 7.2 対応プロセス

```
検知 → 初期対応 → 調査 → 封じ込め → 復旧 → 報告 → 改善
```

### 7.3 顧客への通知

- Critical/Highインシデント: 24時間以内に通知
- 影響を受けた顧客への個別連絡
- インシデントレポートの提供

### 7.4 連絡先

- セキュリティ問題の報告: security@talentsync.example.com
- 緊急連絡: 03-XXXX-XXXX（24時間対応）

### 8.1 個人情報保護法

|------|----------|
| 利用目的の特定・公表 | ○ プライバシーポリシーに明記 |
